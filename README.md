En este ejercicio se nos pide realizar un programa que registre una nueva etapa en la base de datos y genere automaticamente las participaciones de los ciclistas usando consultas SQL, transacciones y operaciones con listas. Empezamos estableciendo la conexion con la base de datos Oracle mediante JDBC y obteniendo el numero de etapa automaticamente usando una consulta SQL que obtiene el valor MAX(numero) de la tabla etapa y le suma 1 para generar el siguiente numero disponible.

A continuacion solicitamos por consola los datos necesarios para crear la etapa, como el origen, destino, distancia en kilometros y la fecha, convirtiendo esta ultima al tipo Date para poder almacenarla correctamente en la base de datos.

Despues iniciamos una transaccion desactivando el autoCommit y realizamos la insercion de la nueva etapa en la tabla etapa mediante un PreparedStatement. Seguidamente eliminamos posibles participaciones anteriores asociadas a esa etapa para evitar duplicados.

El siguiente paso consiste en obtener todos los ciclistas desde la tabla ciclista mediante una consulta SELECT, almacenando sus identificadores en un ArrayList. Posteriormente mezclamos aleatoriamente los ciclistas usando Collections.shuffle y seleccionamos solo los 5 primeros para generar el Top 5 de la etapa, asignando posiciones aleatorias y calculando los puntos correspondientes segun la posicion obtenida.

Finalmente insertamos las participaciones en la tabla participacion para cada ciclista seleccionado y realizamos el commit de la transaccion si todo se ejecuta correctamente. En caso de error se ejecuta un rollback para cancelar todos los cambios y evitar datos inconsistentes. Al terminar, mostramos por pantalla un mensaje indicando que la etapa se ha insertado correctamente junto con su numero, numero de ciclistas y fecha.
